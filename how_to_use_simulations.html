Using the provided simulations

unseen-awg is a resampling-based weather generator, i.e., it simply combines time steps from an existing dataset in a new order. Therefore, simulated time series can be stored very compactly as a “look-up table” into the existing dataset.

Using the simulations requires downloading the “look-up table” (NetCDF) and the dataset of impact-relevant variables that is being resampled.

We provide both at [TODO: insert link/DOI] insert path. As described earlier, all partial zip files of the dataset of impact-relevant variables must be extracted.

generated_time_series = xarray.open_dataset(path_time_series)
ds_impact_relevant_variables = xarray.open_zarr(
    path_ds_impact_relevant_variables
).chunk(
    {"ensemble_member": 1, "init_time": 1}
)  # smaller chunks for reduced memory demand

For each output time step, the time series generated_time_series hold the coordinate vectors of the corresponding sampled atmospheric state in the reforecast dataset, i.e. its ensemble_member, init_timeand lead_time.

The provided dataset includes 500 generated daily weather time series, each 21 years long.

To extract the actual weather data, the generated time series can simply be used as index labels to an xarray dataset of impact-relevant variables

generated_data = ds_impact_relevant_variables.sel(generated_time_series)

The dataset of impact-relevant variables (ds_impact_relevant_variables) is a zarr store; as a result, xarray uses lazy computations, loading only the data into memory that is currently required. dask can be used to parallelize computations. In the simplest case, users can just load desired subsets of the data to memory, e.g.,

generated_data.sel(seed=0).isel(out_time=slice(0, 10)).load()
