Data download and preparation

A set of archived data are available at the World Data Center for Climate: https://www.wdc-climate.de/ui/project?acronym=unseen-awg.

The data are split into two “Experiments”:

  • “An instance of the analog weather generator unseen-awg with precomputed similarities and a large set of generated weather data”

  • “Preprocessed atmospheric circulation and impact-relevant variables from ERA5 and “Extended ensemble forecast hindcast” (ECMWF) for unseen-awg simulations”

Together, these data allow:

  • Using an existing large dataset of unseen-awg simulations. Available impact-relevant variables are: daily minimum, maximum, and mean temperatures, daily total precipitation sums.

  • Generating new simulations.

  • Creating new unseen-awg instances.

No bias correction was applied to these datasets. The provided Snakemake workflow allows applying simple mean bias correction (per lead time, location and day of year) to the temperature variables; implementing more elaborate for bias computation and correction methods is possible.

Data extraction

The archived data consist of NetCDF4 files and Zarr stores. For practical reasons, the Zarr stores are archived as multi-part (split) zip files. When using the Zarr stores, you must unzip all partial files for a given dataset; otherwise, the resulting store will suffer from missing data.

We provide a script unzip_provided_datasets.sh to facilitate unzipping the data. The general syntax is:

./unzip_provided_datasets.sh <dir_containing_zipped_files> <dir_target>

Integrating extracted data into provided Snakemake workflow

The directory workflow/ contains the Snakemake workflow we built for our study. It can be used to generate new weather generators or bias-corrected datasets of impact-relevant variables.

Unzipping the data results in a directory structure and file names that facilitate integration into this workflow (e.g., file names include parameter hashes).

To use the workflow, the unzipped data must be placed in the specific directories defined in configs/paths.yaml (where important base directories are defined, as described here). The weather generator’s directory must be placed in the <dir_wgs> directory, the input datasets in <dir_preprocessed_datasets>, i.e.,

./unzip_provided_datasets.sh <dir_with_partial_weather_generator_files> <dir_preprocessed_datasets>

./unzip_provided_datasets.sh <dir_with_partial_input_files> <dir_wgs>

After extraction, your resulting directory paths should look like this:

<dir_wgs>/wg_reforecasts_5e06172f_f40e9460_1e69bda9
<dir_preprocessed_datasets>/preprocessed_impact_variables_reforecasts/combined_7d1d3d97.zarr
<dir_preprocessed_datasets>/preprocessed_impact_variables_era5/combined_facc0e91.zarr
<dir_preprocessed_datasets>/preprocessed_circulation_reforecasts/combined_5e06172f.zarr
<dir_preprocessed_datasets>/preprocessed_circulation_era5/combined_f3d1f2f7.zarr

There is no prescribed path that the provided unseen-awg simulations must be stored under.
